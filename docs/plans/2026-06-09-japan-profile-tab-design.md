# 日本测试资料生成器 Tab 设计

## 目标

在现有 `ChatGPT 提取支付链接` 页面中增加顶部 Tab：

- 支付链接控制台
- 日本测试资料

日本测试资料页面通过本地同源接口读取内置生成器，提供随机生成、关键词筛选、字段复制与 TXT 下载，不修改现有支付网关提链流程。

## 本地接口

浏览器同源入口：

```text
GET /api/japan-profile
GET /api/japan-profile?keyword={keyword}
```

浏览器只请求本地 `/api/japan-profile`；后端使用内置日本地区、姓名、公司和设备资料池随机组合，并在返回前执行白名单过滤。该功能不依赖外部接口、跨域策略或公网网络。

## 数据边界

页面展示并保留：

- 汉字姓名
- 平假名和片假名
- 性别、生日
- 固定电话、手机、邮箱
- 都道府县、市、邮编、完整地址
- 公司、薪资
- 操作系统、User-Agent、主页

生成器不生成以下字段；映射层也会继续丢弃这些字段，不渲染、不复制、不下载：

- `nationalid`
- `credit_card_info`
- `password`
- `security_question`

## 页面结构

Tab 导航位于现有 Header 下方，作为全局视图切换控件。

支付链接控制台：

- 保持现有 DOM、表单和行为不变。
- 作为默认 Tab。

日本测试资料：

- 顶部工具栏包含关键词输入、搜索、换一组、TXT 下载。
- 主结果区按“基本信息、地址信息、工作与设备”分组。
- 每个字段提供独立复制按钮。
- 加载、成功、错误状态在工具栏下方统一显示。
- 首次进入该 Tab 时自动请求一组数据；后续切换不重复请求，除非用户主动刷新。

## 数据流

1. 用户首次进入日本测试资料 Tab。
2. 前端向 `/api/japan-profile` 发送 GET 请求。
3. 后端按关键词筛选本地地区资料池并生成白名单字段。
4. 前端校验响应并映射白名单字段。
5. 页面渲染结果。
6. 搜索时将关键词编码后调用同源接口。
7. TXT 下载只导出当前白名单字段。

## 错误处理

- 本地接口错误：保留上一次成功结果并显示错误提示。
- HTTP 非 2xx：显示接口状态码。
- JSON 结构缺失：缺少字段显示 `—`，不回退到敏感或猜测数据。
- 搜索关键词为空：不发送请求并提示输入关键词。
- 不把接口原始 JSON写入页面、日志或本地存储。

## 样式

- 沿用现有蓝色、白色、浅灰产品视觉。
- 使用紧凑工具栏和分组字段列表，避免与现有控制台形成两个不同产品。
- 桌面端采用多列布局，移动端切为单列。
- Tab、按钮和复制控件保持稳定尺寸，避免加载时布局跳动。

## 验证

- 静态检查：HTML、CSS 和 JavaScript 无语法错误。
- 浏览器验证：两个 Tab 可切换，原控制台仍可使用。
- 接口验证：随机生成和关键词搜索可本地返回并渲染白名单字段。
- 隐私验证：DOM、TXT 和复制控件中不存在敏感字段。
- 响应式验证：桌面与移动视口无溢出、遮挡或文字重叠。
